pgAdmin FAQ

  • When I run pgAdmin I get an error like: "Run-time error '430': Class does not support Automation or does not support expected interface."

    Try re-installing the latest version of MDAC which is available from http://www.microsoft.com/data

 

  • When I run pgAdmin I get an error like: "Run-time error '3706': Provider cannot be found. It may not be properly installed.."

    Try re-installing the latest version of MDAC which is available from http://www.microsoft.com/data

 

  • I get errors like: "pgadmin_desc does not exist" and "The rowset does not support fetching backwards" when browsing columns in the table browser, also, every time I log in, pgAdmin trys to create a table called "pgadmin_desc".

    In the DSN configuration, make sure the Use Declare/Fetch option is not checked.

  • Where do I get support?

    Try either the pgAdmin Support mailing list - pgAdmin-support@greatbridge.org or the pgAdmin project site at http://www.greatbridge.org/project/pgadmin/. Many users also read the pgsql-interfaces mailing list (see http://www.postgresql.org). We cannot guarantee that we can help or even have the time to try but we will do our best.

     

  • Where are new releases announced?

    You can subscribe to the pgAdmin-announce mailing list at http://www.greatbridge.org/project/pgadmin/ to get announcements as they are made. They will generally be CC'd to the pgsql-interfaces list as well.
