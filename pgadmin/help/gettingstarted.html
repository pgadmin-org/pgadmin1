pgAdmin Getting Started Guide

Contributed by: francl with updates from Jean-Michel Poure & Dave Page

I spent a lot of time trying to setup pgAdmin to run under Windows and thought this might help some newbies like me. Anyway here is what I did !

  1. IMPORTANT: Backup your system in case something goes wrong.
  2. If you are running on Windows 95
  3. Install the MDAC 2.6 RTM (not the Software development kit). http://www.microsoft.com/data/download_260rtm.htm
  4. Get the PostgreSQL ODBC driver: ftp://ftp.greatbridge.org/pub/pgadmin/stable/psqlodbc-07_01_0003.zip. Copy the contents of the archive (ie. psqlodbc.dll) to you Windows System directory.
  5. Get the PostgreSQL ODBC driver registry settings: ftp://ftp.greatbridge.org/pub/pgadmin/stable/psqlodbc-registry.zip. Unzip the enclosed Regedit file and import it to the registry by double clicking it.
  6. Download and install the "Windows Installer" file "InstMsi.exe" ver1.1 from microsoft. This will be used to install the "pgAdmin" software after you have unzipped it: http://www.microsoft.com/downloads/search.asp 
  7. Unzip the latest "pgAdmin" software and you will get a file "pgAdmin.msi". You should be able to double click this to install it.
  8. Make sure you have the postmaster running on your PostgreSQL server in order to connect. Either ensure that the postmaster process was started with the -i option, or, new PostgreSQL's have a data/postgresql.conf file. Open the file in you preferred editor and uncomment #tcpip_socket=false and write tcpip_socket=true. You will need to restart PostgreSQL for this change to take effect.
  9. You may have to add a line to your "data/pg_hba.conf" file on your server to allow your windows PC to connect to the PostgreSQL Database. See below for sample line I added. Just replace 192.168.1.1 with your windows PC's IP address. You can allow access from a group of hosts or subnet by adjusting the subnet mask as appropriate.
    10. host 192.168.1.1 255.255.255.255 password
  10. Once all this is down you can start pgAdmin and add a new datasource from the Login Prompt, or use the ODBC data source administrator in the Control Panel.
  11. Once you are running pgAdmin, go into Utilities and try to run "psql". If it doesn't run, make sure the full path name to the "psql.exe" is specified (should be something like "C:\Program Files\pgAdmin\psql.exe")
