pgAdmin Getting Started Guide

Contributed by: francl with updates from Jean-Michel Poure & Dave Page

This document describes how to install pgAdmin stable release.
WARNING: Backup your system in case something goes wrong.

1) Upgrading your Windows system

  1. Install the MDAC 2.6 RTM (not the Software development kit) from:
    http://www.microsoft.com/data/download_260rtm.htm
  2. If you are not running Access2K, jump to step 3.
    If you are running Access2K, download and install JetSP3 from:
    http://www.microsoft.com/data/download_Jet4SP3.htm
  3. Download and install the "Windows Installer" file "InstMsi.exe" ver1.1 from microsoft:
    http://www.microsoft.com/downloads/search.asp 
  4. Get the PostgreSQL ODBC driver: the recommended versions are now in:
    ftp.postgresql.org/pub/odbc/versions/full or ftp.postgresql.org/pub/odbc/versions/msi.

2) Installing pgAdmin

a) Getting pgAdmin latest stable release

  1. Go to http://www.greatbridge.org/project/pgadmin/download/download.php
    and download the last stable release installer.
  2. Unzip the latest "pgAdmin" software and you will get a file "pgAdmin.msi".
    You should be able to double click this to install it.

b) Configuring PostgreSQL Server to accept TCP/IP connections

  1. Make sure you have the postmaster running on your PostgreSQL server in order to connect. Either ensure that the postmaster process was started with the -i option, or, new PostgreSQL's have a data/postgresql.conf file. Open the file in you preferred editor and uncomment #tcpip_socket=false and write tcpip_socket=true. You will need to restart PostgreSQL for this change to take effect.
  2. You may have to add a line to your "data/pg_hba.conf" file on your server to allow your windows PC to connect to the PostgreSQL Database. See below for sample line I added. Just replace 192.168.1.1 with your windows PC's IP address. You can allow access from a group of hosts or subnet by adjusting the subnet mask as appropriate.

    3. host 192.168.1.1 255.255.255.0 password

  3. Create a PostgreSQL user with CREATEDB and CREATEUSER privileges.

c) Configuring ODBC connection

  1. Once all this is down you can start pgAdmin and add a new datasource from the Login Prompt, or use the ODBC data source administrator in the Control Panel.
  2. Once you are running pgAdmin, go into Utilities and try to run "psql". If it doesn't run, make sure the full path name to the "psql.exe" is specified (should be something like "C:\Program Files\pgAdmin\psql.exe")
