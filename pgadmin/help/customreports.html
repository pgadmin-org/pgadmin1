pgAdmin Custom Reports

The pgAdmin Report Browser has an 'Add' button that allows you to add custom reports to pgAdmin. The following steps show you (the user) what is required to achieve this. To create new reports you will require Seagate Crystal Reports v7 or v8. Newer versions have yet to be tested.

  1. Create a new report in Crystal Reports - specify an 'Active Data' data source using either ADO or RDO (ADO has been known to crash Crystal, but RDO seems OK).
  2. Select a DSN and then Add either the RDO or ADO table to the report.
  3. You will be prompted to either select a Table or View, or enter an SQL query. Use an SQL query, and copy it into Notepad for reference later.
  4. Design the report as required.
  5. Under File -> Summary Info, set the Author, Comments, Title and Subject. These fields are used by pgAdmin to display the Browse Info. 
  6. In the Template field, paste/type the SQL query you entered earlier (without any <CR>'s), or SELECT * FROM <Table>/<View>. The is used by pgAdmin to retrieve the data for the report.
  7. Save the report.
  8. In the pgAdmin Report Browser, select Add, then click the '...' button to browse for your new report. The default value for each field will be extracted from the report. You may want to alter values to allow the use of the same .rpt file for multiple reports.
  9. If you have used either pgadmin_table_cache or pgadmin_seq_cache in your report, click the relevant 'Refresh...' option to ensure they are updated.
  10. If the Report has Groups defined, you may want to check 'Show Group Tree...' which will default the Group Tree in the report viewer to On. It may still be toggled by the user at runtime.
  11. Click 'Add Report' and away you go!
  12. If you write any useful reports that others may benefit from feel free to email them to me for inclusion if you like! All will be gratefully received!
