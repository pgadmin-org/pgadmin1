pgSchema

pgSchema is an object library that will represent an entire PostgreSQL framework, not just a single database. It will provide collections of objects like

  • Projects
  • Repositories
  • Databases
  • Users
  • Types
  • Tables
  • Packages
  • Functions
  • Triggers
  • Views
  • Logs

It is a hierarchical object model with methods and events in place to provide a framework for any application (not just pgAdmin) for logging, query execution and browsing/creating/deleting any database object (ACLs allowing of course).

The intention is to replace the current Browsers in pgAdmin with a single Treeview Browser similar (though superior :-)) to the one found in the MS SQL Server Enterprise Manager.

Full Development Environment

Our intention is to define PostgreSQL standard development framework. We plan to implement it in pgAdmin with the help of the PostgreSQL community.

Repositories framework

A repository is a dedicated PostgreSQL database. Repositories organize information horizontaly (on different machines) and vertically (development workfow from design to implementation and backup). This answers the needs of organizations running a complex framework of postgreSQL servers.

  • Central repository
    • source code
    • project information
    • loggin system
  • Backup repositories
    • backup data, source code and logs
    • possibly with the use of one-way replication
  • Log servers
    • pgAdmin user log
    • PostgreSQL log
    • Libraries to log information form other applications (at first Java, PHP and VB).
  • Development servers
  • Production servers
